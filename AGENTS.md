# Project Guide

## Tech Stack & Package Manager

- **Framework:** Waku (React 19 RSC) + Vite
- **Language:** TypeScript (strict, esnext/bundler)
- **Routing:** File-based (`src/pages/`), types from `waku-jotai/router`
- **Styling:** Tailwind CSS v4, `tw-animate-css`, `cva`
- **UI:** `@base-ui/react` primitives composed into `src/components/ui/` (shadcn-style)
- **State:** Jotai — all atoms in `src/lib/state/`
- **Package manager:** pnpm (never npm or yarn)

> **Never edit `pages.gen.ts`** — auto-generated by `pnpm typegen`; edits will be overwritten.

## Commits

Conventional Commits — `<type>[scope]: <desc>` with optional body. Types: `feat | fix | refactor | chore | docs | style | test | perf`. Lowercase, no trailing period, body wrapped at 72 chars.

## Linting & Formatting

- **Rules:** `curly`, `eqeqeq`, `no-floating-promises`, `promise/always-return` (`.then()` must `return;`), `exhaustive-deps: warn`, `no-unused-vars: error` (prefix unused with `_`)
- **Formatter:** tabs, 120 width, trailing commas, Tailwind class sorting
- **Ignored:** `dist/`, `.cache/`, `.vercel/`, `pages.gen.ts`, `node_modules/`
- **Disabled in oxlintrc:** `no-explicit-any`, `no-empty-object-type`, `no-unsafe-type-assertion`

## Conventions

- `@/` → `./src/`
- React 19 — no JSX import
- No default exports except page components
- Routes: `src/pages/<route>/index.tsx`
- `cn()` util: `clsx` + `tailwind-merge` in `src/lib/utils.ts`
- `cva()` for variants; PascalCase components in `src/components/ui/`
- State atoms in `src/lib/state/<domain>.ts` — `atom()` for local, `atomWithStorage()` for persisted; never `createContext`

## Server vs Client Components (RSC Boundary)

Waku uses React 19 RSC. Getting the boundary wrong is the most common source of bugs.

**Server Components** (default — no directive needed):

- Can be `async`, `await`, import server-only modules, access DB/cache/storage/env
- Cannot use hooks, event handlers, or browser APIs

**Client Components** (`"use client"`):

- Required for: hooks, event listeners, browser APIs, Jotai atoms
- Push `"use client"` as far down the tree as possible — keep them leaf nodes
- Cannot be `async`; cannot import server-only modules

**Server Actions** (`"use server"`): use for form submissions and mutations.

Rule of thumb: if it uses a Jotai atom or handles interaction → `"use client"`. Pass server-fetched data down as props — don't re-fetch in the client. All three singletons (`getDb`, `getCache`, `getStorage`) are server-only.

## Infrastructure

- **DB:** Postgres via Drizzle ORM (`pg` driver), schema in `src/lib/db/schema.ts`, `getDb()` lazy singleton
- **Cache:** Redis via ioredis, `getCache()` — `get`, `set`, `delete`, `clear`, `has`
- **Storage:** S3-compatible via `@aws-sdk/client-s3`, `getStorage()` — `write`, `read`, `delete`, `list`, `exists`, `clear`, `url`
- `.env` gitignored; copy `.env.example` for defaults

> All three (`getDb`, `getCache`, `getStorage`) are server-only. Never import in a Client Component.

## Auth & Sessions

- Session data is server-only — never expose tokens to the client
- Read auth state in Server Components/Server Actions; pass minimal user data as props
- Do not store auth tokens in Jotai atoms

## Waku Routing

File-based in `src/pages/`. Pages export default async component + `getConfig`. `render: "static"` (default) or `"dynamic"` (default for API handlers).

### Page props: `PageProps<'/path'>`

| File                      | Prop                 |
| ------------------------- | -------------------- |
| `index.tsx`               | `path`, `query`      |
| `[slug]/index.tsx`        | `slug`               |
| `[...catchAll]/index.tsx` | `catchAll: string[]` |

### API routes

In `_api/` dir — `_api` prefix stripped from URL. Export HTTP method functions:

```ts
export const POST = (req: Request) => Response.json({ ok: true });
export async function GET(_req: Request, { params }: ApiContext<"/users/[id]">) {
	return Response.json({ id: params.id });
}
```

### Other

- `_layout.tsx` wraps children; `_root.tsx` customizes `<html>`
- `(group)/` dirs organize without affecting URLs
- Slices in `_slices/` with independent render modes (`<Slice id="..." />`)
- Navigation: `<Link>` from `waku`, `useRouter()` for programmatic
- Metadata auto-hoisted to `<head>`
